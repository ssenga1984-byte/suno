# JUON サイト ロールバック手順書

管理画面からの編集で問題が起きた場合に、直前の正常な状態に戻す手順です。

## 方法 1: Netlify コンソールから戻す（推奨）

1. [Netlify](https://app.netlify.com/) にログイン
2. 対象サイトのダッシュボードを開く
3. 左メニュー「**Deploys**」をクリック
4. デプロイ一覧から、**問題が起きる前の正常なデプロイ**（緑色の「Published」マーク付き）を探す
5. そのデプロイをクリックして詳細画面を開く
6. 「**Publish deploy**」ボタンをクリック
7. 確認ダイアログで「**Publish**」を押す
8. サイトが直前の状態に戻ったことをブラウザで確認する

> デプロイ一覧は時系列で並んでいるので、「いつ頃正常だったか」がわかれば見つけやすい。

## 方法 2: GitHub でコミットを戻す

1. [GitHub リポジトリ](https://github.com/satoshi19840417/suno) を開く
2. 「**Commits**」から、問題のコミット（CMSからの自動コミット）を特定する
3. そのコミットの「**Revert**」ボタンを使って revert コミットを作成する
4. revert がマージされると Netlify が自動ビルド・公開する

## 注意事項

- Netlify のデプロイ履歴は過去のビルド結果を保持しているため、方法 1 は即座に反映される
- 方法 2 は Git 履歴に revert 記録が残る（追跡性が高い）
- いずれの場合も、管理画面側のコンテンツ JSON は自動では戻らない。方法 2 を使うとコンテンツも含めて Git 上で戻る
- 判断に迷う場合は開発者に連絡する

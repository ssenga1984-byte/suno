## 1. 目的

現在のWebサイトは、主要ページをReact/TailwindのDOMで実装し、下層ページの一部は `public/assets/mockups/*.png` を暫定表示として残す構成になっている。既存の写真は雰囲気づくりには使えているが、活動ごとの違いを直感的に理解するには不足している。

本計画では、PDF `202605.pdf` に含まれる実活動写真を活動種別ごとに整理し、サイト上で「どの活動が自分に合いそうか」を判断しやすくする。

## 6. 実装方針

### Phase 1: 写真選定と切り出し

1. `202605_activity_photo_focus_board.png` を見ながら、活動カテゴリごとに採用候補を選ぶ。
2. 顔が大きく写る写真、個人名やオンライン画面が判別できる写真は除外またはトリミングする。
3. 採用画像を `public/assets/photos/activities/` に保存する。
4. ファイル名は用途が分かる英語名にする。

例:
- `forest-school-kaminosen.png`
- `forest-volunteer-pruning.png`
- `field-school-ringo.png`
- `field-school-kaki.png`
- `course-workshop-kansai.png`
- `regional-town-walk.png`

### Phase 2: データ構造の追加

`src/data/siteContent.ts` に活動写真用のデータを追加する。

候補:
- `activityPhotoGroups`
- `activityMedia`
- `routePhotoHighlights`

必要フィールド:
- `id`
- `title`
- `category`
- `src`
- `alt`
- `source`
- `recommendedRoutes`
- `safetyNote`

### Phase 3: `#/activities` に写真付き活動比較を追加

最初に追加すべき場所:
- `ActivityEntrySection`
- `ProgramOverview`
- `PermanentInitiativesSection` の前後

推奨構成:
- 森林の楽校
- 田畑の楽校
- 地域の活動
- 講座・はじめての入口
- 樹恩割り箸

このページは活動選択の中心なので、PDF写真の反映効果が最も高い。

### Phase 4: 詳細ページへ展開

次に以下へ展開する。

- `#/forest-school`: 森林作業写真を2-3枚
- `#/field-school`: 農作業・収穫・食の写真を2-3枚
- `#/regional`: 地域散策・世話人会の写真を2枚
- `#/course`: 入門講座写真を1-2枚。DOM化と同時が望ましい

### Phase 5: 募集カードへの反映

`RecruitmentItem` に `image` または `categoryImageId` を持たせるか、カテゴリごとに共通サムネイルを割り当てる。

推奨:
- 個別イベント写真を毎回持たせるのではなく、まずはカテゴリ共通画像で運用する
- 公式募集内容と写真が一致しない誤解を避けるため、「写真は活動イメージ」と明示できる設計にする

## 7. 注意点・リスク

1. 肖像権・許諾
   - PDF掲載済みでも、Webサイトで再利用できるとは限らない。
   - 顔が判別できる写真は、許諾確認または顔なし加工が必要。

2. 写真と募集内容の不一致
   - 過去活動写真を現在募集中イベントに直接紐づけると誤解が出る。
   - 募集カードでは「活動イメージ」として扱う。

3. 既存サイト契約との整合
   - 現在は主要ページDOM、下層の一部はPNG暫定表示という設計。
   - 写真追加で `#/course` や `#/voices` をDOM化する場合は、`DESIGN.md` と `docs/qa-checklist.md` も更新する。

4. アクセシビリティ
   - すべての写真に意味のある `alt` を設定する。
   - 写真上に文字を置く場合は、十分なコントラストを確保する。

5. パフォーマンス
   - PDFから切り出した高解像度画像をそのまま使わない。
   - Web用に横幅1200px前後、カード用は600-800px程度へ圧縮する。

## 8. 推奨する最初の作業単位

最初の実装は `#/activities` に限定する。

理由:
- 活動カテゴリの違いを見せる効果が最も大きい
- 既にDOMページなので、写真追加の効果を検証しやすい
- `#/forest-school` や `#/field-school` へ後から展開しやすい

最初に作るもの:
1. `public/assets/photos/activities/` に活動別写真を6-8枚保存
2. `src/data/siteContent.ts` に写真メタデータを追加
3. `ActivitiesPage` に「活動写真で見る」セクションを追加
4. Desktop `1440x900` / Mobile `390x844` で横スクロール・文字被り・写真の見切れを確認
5. `npm run build` を通す

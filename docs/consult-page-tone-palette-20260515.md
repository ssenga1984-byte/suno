# 相談するページ カラーパレット・トンマナ案

## リクエスト整理

対象は JUON NETWORK サイトに新設する `#/consult` 想定の「相談する」ページ。
主な利用者は、初参加者、学生、個人参加を検討している人、親子参加を検討している人、企業・学校・団体の担当者、受け入れ先の地域・団体、報道関係者、支援や連携の入口を探している人。

目的は、問い合わせ前の不安や迷いを整理し、「活動を探す」「問い合わせページへ進む」「公式窓口へ進む」のどれを選べばよいか判断しやすくすること。

既存制約:
- 既存の `forest` パレットをブランドの軸にする。
- 架空のイベント、日程、費用、参加条件を作らない。
- ローカルサイトが個人情報を保存・送信するように見せない。
- 最終送信は公式問い合わせ窓口へ誘導する。

## 方向性オプション

### A. 森の安心ガイド

落ち着いた森の緑をベースに、初参加者の不安を一つずつほどく案。
見出しは日本語 serif で信頼感を出し、本文と相談カードは sans-serif で読みやすくする。

強み:
- 既存サイトのトンマナと最も自然につながる。
- NPOらしい信頼感を崩さない。
- 相談ページが「営業的」になりにくい。

リスク:
- 配色が既存ページと近くなりすぎると、新設ページとしての役割差が弱い。

### B. 若葉と空のはじめて導線

若葉グリーンと淡い空色を補助色にして、軽さと入りやすさを強める案。
不安解消カード、相談カテゴリ、次の行動ボタンを明るく見せる。

強み:
- 「まずは相談してよい」という心理的ハードルを下げやすい。
- 初参加者、親子、若い層に向く。
- 問い合わせページとの差分を作りやすい。

リスク:
- 明るくしすぎると、JUON の信頼感や自然の落ち着きが薄くなる。

### C. 事務局コンシェルジュ

白地と濃い緑を多めに使い、相談カテゴリを整理された案内窓口のように見せる案。
学校・企業・団体連携や取材にも対応しやすい。

強み:
- 相談内容の分類が明確。
- 団体・企業・学校の担当者に伝わりやすい。

リスク:
- 初参加者には少し事務的に見える。
- 問い合わせページとの差分が弱くなる。

## 推奨方向

推奨は **B. 若葉と空のはじめて導線**。

理由:
- 相談ページは、問い合わせより前の「迷いを受け止める」役割を持つため、既存の問い合わせページより少し明るく、入りやすい印象が必要。
- ただしブランドから外れないように、ベースは `forest.deep`, `forest.leaf`, `forest.linen` を維持する。
- 空色と若葉色は、ページ全体を支配する色ではなく、相談カテゴリ、安心メモ、選択肢の区別に限定して使う。

| 用途 | 色名 | 値 | 使い方 |
|---|---|---:|---|

- `forest.gold` は主要CTAだけ。相談カテゴリごとに金色ボタンを乱用しない。
- `consult.alert` はエラー色ではなく「公式で最新確認」などの注意にだけ使う。

## トンマナ

### キーワード

- 迷っていても大丈夫
- 初参加の前に確認できる
- 押しつけない
- 自分に合う入口を選べる
- 最終確認は公式窓口へ

### 避ける印象

- 申し込みを急がせる
- 相談フォームで個人情報を送信できるように見せる
- イベント内容や参加条件を断定しすぎる
- 派手なボランティア募集広告のように見せる

### 文体

短く、安心を先に置く。

例:
- `どの活動が合うか迷ったら、まず相談できます。`
- `参加前の不安を整理してから、公式窓口へ進めます。`
- `親子参加、団体連携、支援の関わり方も確認できます。`

避ける:
- `今すぐ申し込みましょう`
- `誰でも必ず参加できます`
- `このページから問い合わせを送信できます`

## ページ構成

### 1. ファーストビュー

役割:
相談してよい理由を最初に伝える。

構成:
- Eyebrow: `Consult`
- H1: `迷ったら、まず相談できます。`
- Lead: `参加前の不安、親子参加、団体連携、支援の関わり方を整理して、次に進む入口を選べます。`
- Primary CTA: `相談内容を選ぶ`
- Secondary CTA: `問い合わせページへ`

背景:
- 顔が写らない森や道の写真。
- 既存 Standard Banner 系を使い、相談ページ専用の派手なヒーローは作らない。

### 2. 相談対象を選ぶ

最初に「自分はどの立場か」を選べる構成にする。
相談ページは問い合わせ分類より前の段階なので、事務的な種別名だけでなく、利用者が自分ごと化しやすい言葉を添える。

推奨カテゴリ:

| 対象 | 表示名 | 主な相談内容 | 次の導線 |
|---|---|---|---|
| 学生向け | 学生として参加したい | 初参加、友人同士、授業・ゼミ・サークル、交通や持ち物、活動証明の確認 | `#/first-time`, `#/events`, `#/contact` |
| 個人向け | 個人で参加したい | 一人参加、親子参加、体力面、年齢、日帰り参加、どの活動が合うか | `#/first-time`, `#/events`, `#/contact` |
| 企業向け | 企業・団体で関わりたい | CSR、社員参加、研修、協賛、継続支援、団体参加の相談 | `#/support`, `#/contact` |
| 学校向け | 学校・ゼミ・教育活動で関わりたい | 授業連携、体験学習、学生団体、ゼミ活動、受け入れ可否の確認 | `#/contact` |
| 受け入れ先向け | 地域・受け入れ先として相談したい | 地域活動の受け入れ、協働、活動場所、運営協力、募集掲載の相談 | `#/contact` |
| 報道向け | 取材・掲載について相談したい | 取材、写真利用、掲載確認、広報連携、活動紹介 | `#/contact` |
| 支援者向け | 寄付・会員・支援を相談したい | 寄付、会員、物品支援、継続支援、支援方法の確認 | `#/support`, `#/contact` |

ページ上では全カテゴリを横並びに詰め込まず、優先表示は 6 件までに抑える。
候補は「学生」「個人」「企業・団体」「学校」「受け入れ先」「報道・支援」のようにまとめ、詳細はカード内で補足する。

### 3. よくある相談事項

カードは 6 件程度。

- 初参加でも大丈夫か知りたい
- 親子や子ども連れで参加できるか知りたい
- 体力や持ち物が不安
- どの活動を選べばよいかわからない
- 学校・企業・団体として関わりたい
- 地域・受け入れ先として協働できるか知りたい
- 取材や掲載について確認したい
- 寄付・支援・会員参加を検討したい

カードの動き:
- クリックで問い合わせページへ送るのではなく、まず下部の「次の行動」へ視線を流す。
- カード内に断定的な回答を詰め込みすぎない。

### 4. 相談前に整理すること

3ステップ:
1. 参加したい地域や時期
2. 学生、個人、企業、学校、受け入れ先、報道などの立場
3. 確認したい不安や条件

ここは `consult.sky` の淡い背景で、問い合わせページとの橋渡しにする。

### 5. 次の行動を選ぶ

3つの導線:
- `募集中の活動を見る` -> `#/events`
- `はじめての方へ` -> `#/first-time`
- `問い合わせページへ進む` -> `#/contact`

問い合わせページへ進むカードには、次の説明を添える。
`問い合わせページでは、公式窓口へ進む前に内容の種類を整理できます。`

## 問い合わせページとの役割分担

相談ページ:
- 不安や目的を整理する。
- 自分に合う入口を探す。
- まだ問い合わせ文が決まっていない人向け。
- 学生、個人、企業、学校、受け入れ先、報道などの立場別に入口を分ける。

問い合わせページ:
- 連絡内容を分類する。
- 公式問い合わせ先へ進む。
- すでに聞きたいことがある人向け。
- 相談ページで選んだ立場を、問い合わせ内容の整理に引き継げるようにする。

問い合わせページ側には「よくある問い合わせ内容」を追加するとよい。

候補:
- 活動参加について
- 参加条件、持ち物、当日の流れ
- 学生・学校連携
- 企業・団体連携
- 地域・受け入れ先からの相談
- 取材・広報掲載
- 会員・寄付・支援
- 資料・公開情報・その他

## タイポグラフィ

- Display: 日本語 serif。H1 と大見出しだけ。
- Heading: sans-serif 太字。相談カテゴリやカード見出し。
- Body: sans-serif。16px 前後、行間広め。
- Caption: muted color。公式確認、保存しない注意、補足説明。

フォントサイズは viewport 幅連動で拡大しない。
モバイルでは見出しを短くし、カード内テキストの詰め込みを避ける。

## 実装引き継ぎメモ

- 新ルート候補: `#/consult`
- ナビ/ヘッダーCTA: `相談する`
- 既存 `#/contact` は `お問い合わせ` のまま維持。
- `src/data/siteContent.ts` に consult 用ページデータを追加する場合、問い合わせページと文言重複しすぎないようにする。
- `tailwind.config.ts` に追加するなら `consult` カラートークンを `forest` の補助として追加する。
- `DESIGN.md` には `#/consult` を DOM 正本に追加し、Contact との役割分担を追記する。
- 実装後は `npm run build`、`#/consult` と `#/contact` の desktop/mobile 表示確認、`document.title`、`alt`、`aria-label`、外部リンクを確認する。

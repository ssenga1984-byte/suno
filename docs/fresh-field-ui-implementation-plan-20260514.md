# Fresh Field UI 実装計画書

## 目的

既存の JUON NETWORK サイトを、若者・初参加者が「自分も参加できそう」と判断しやすい UI に改修する。生成済み UI モックを視覚リファレンスにしつつ、実在する活動内容・日程・写真・募集状況は既存データと公式リンクだけを根拠に扱う。

## 絶対条件

- 技術構成は React + Tailwind CSS のまま維持する。
- 対象ページは `#/`, `#/first-time`, `#/activities`, `#/events`, `#/forest-school`, `#/contact`。
- 実在の活動名、日程、地域、公式リンク、募集注記は `src/data/siteContent.ts` の既存データを使用する。
- 架空のイベント、開催地、参加人数、費用、空き状況、活動写真を追加しない。
- 生成済み画像は UI 方向性の参照として扱い、特定イベントの実写真として見せない。
- アイコンは lucide-react の透過 SVG を使用する。
- `document.title`, `alt`, `aria-label`, 外部リンク、キーボードフォーカスを維持する。
- 完了前に `npm run build` とデスクトップ/モバイル表示確認を行う。

## トンマナ

「信頼できる日本の NPO」+「初参加でも入りやすい屋外体験」+「参加前の不安を先回りして解消する」方向。

- 深い森の緑: 信頼感、ヘッダー、ヒーロー背景。
- 若葉グリーン: 初参加・安心バッジ、補助導線。
- 空色: 不安解消パネル、案内カード。
- 日差しイエロー: 主要 CTA。
- 土色: 地域・フィールド感の補助。
- 赤みのアクセント: 締切や注意喚起のみに限定。

## ページ別方針

- トップ: 森のフルブリードヒーローを維持し、「参加できる活動を見る」「はじめての方へ」を第一導線にする。初参加 OK、1日参加あり、一人参加 OK、持ち物確認の安心バッジを追加する。
- はじめて: 参加前の不安を減らすページとして、当日の流れ、持ち物、体力目安、公式確認の導線を整理する。
- 活動一覧: 募集中スナップショットを先に見せ、活動種別カードへつなぐ。実データ以外の募集情報は追加しない。
- イベント: 既存の月別・地域別募集データを見やすく整理し、公式一覧への確認導線を残す。
- 森林の楽校: 公式参考ページを踏まえ、下刈り、間伐、自然散策、地域交流、持ち物を一般化して説明する。各回の詳細は公式ページ確認を前提にする。
- 問い合わせ: ローカル送信フォームに見える UI をやめ、公式問い合わせ・募集一覧・参加案内へのルーティングカードに置き換える。

## アニメーション

- ヒーロー、カード、補助パネルに控えめなフェード/ライズ演出を付ける。
- CTA とカードの hover/focus は小さな移動・色変化に限定する。
- スクロール妨害や内容を隠す演出は入れない。
- `prefers-reduced-motion` を尊重する。

## Worker 分担

- Worker A: 画像参照素材、Tailwind トークン、設計ドキュメント。
- Worker B: `src/App.tsx` の主要 UI 実装とビルド確認。
- Worker C: ビルド、事実保持、アクセシビリティ、モバイル表示のレビュー。
- Orchestrator: 各 Worker 成果物を確認し、ビルド失敗・表示崩れ・文字切れを検出した場合に修正を差し戻す。
